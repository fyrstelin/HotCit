RESTful API

Get standard content

Get All Characters

GET /characters/
Response
200 OK - Character[ ]
Note

Set the Accept header to "text/plain" to get an array of strings.

Get Character

GET /characters/{charactername}/
Response
200 OK - Character
404 Not found (no such character)
Note

Set the Accept header to "image/jpeg" to get the image for the character.

Get All District

GET /districts/
Response
200 OK - District[ ]
Note

Set the Accept header to "text/plain" to get an array of strings.

Get District

GET /districts/{districtname}/
Response
200 OK - District
404 Not found (no such district)
Note

Set the Accept header to "image/jpeg" to get the image.

Get your hand

404 Not Found (game does not exists)

Get your options

GET /games/{gamename}/my/options/
Response
200 OK - Option[ ]
404 Not Found (game does not exists)

Get players

GET /games/{gamename}/players/
Response
200 OK - Player[ ]
404 Not found (game does not exists)
Note

Set the Accept header to "text/plain" to get an array of strings.

Get player

GET /games/{gamename}/players/{username}/
Response
200 OK - Player
404 Not found (game does not exists)
404 Not found (no such player in game)

Get districts for a player

OBJECTS

Get discarded characters

GET /games/{gamename}/dicarded/
Response
200 OK - District[ ]
404 Not found (game does not exists)

Get updates

GET /games/{gamename}/updates/{lastSeenUpdate}/
Response
200 OK - Update[ ]
204 No content (No newer updates)

Note

If the server estimates there is to many updates, it will send one update with message "Update all"

Game setup

Create game

POST /games/new/
Data
GameData
Response
201 Created
409 Conflict (game already exists)

Retrieve list of active games

GET /games/
Response
200 OK - GameInfo[ ]

Join game

PUT /games/{gamename}/join/
Data
Optional password
Response
204 No Content (game joined)
304 Not Modified (player is already in game)
403 Forbidden (wrong password)
404 Not found (game does not exists)
409 Conflict (the game is full)

Leave game

DELETE /games/{gamename}/join/
Response
204 No Content (game left)
304 Not Modified (player is not part of the game)
404 Not Found (game does not exists)

"I am ready"

PUT /games/{gamename}/ready/
Response
204 No Content (you are marked as "ready")
304 Not Modified (player is already marked as "ready")
404 Not Found (game does not exists)

"I am not ready"

DELETE /games/{gamename}/ready/
Response
204 No Content (you are marked as "not ready")
304 Not Modified (player is already marked as "not ready")
404 Not Found (game does not exists)
409 Conflict (player is not a part of the game)

Playing the Game

These requests requires authorization and can return a "401 Unauthorized". Futhermore they will return a "403 Forbidden" if you are not following the protocol.

Choose character

PUT /games/{gamename}/choose/character/{charactername}/
Response
204 No content (success)
404 Not found (the character is not in play)
404 Not found (the game does not exists)
410 Gone (the character is already choosen by another player)
Note

This call should be called when you get an option to choose a character.

Use an action (gold)

PUT /games/{gamename}/action/gold/
Response
204 No content (success)
404 Not found (the game does not exists)

Use an action (get districts to choose from)

PUT /games/{gamename}/action/draw/
Response
204 No Content
302 Found - Location: /games/{gamename}/my/options/
404 Not found (the game does not exists)
Note

The available districts can be found by calling /games/{gamename}/my/options/

Select district

PUT /games/{gamename}/action/draw/{district}
Response
204 No Content (success)
404 Not found (the game does not exists)
Note

This call should be called when you get an option to choose a district

Build a district

PUT /games/{gamename/action/build/{district}
Response
204 No Content (build)
402 Payment Required (you do not have enough gold)
404 Not Found (You don't have the district)
404 Not found (the game does not exists)

Use ability

PUT /games/{gamename}/ability/
Data
AbilityInfo
Response
204 No Content (ability used)
404 Not found (the game does not exists)

Examples

GameStateVector

GameStateVector
Description"Waiting for 'afk' to take an action or use ability"
Round5
Step2
Turn"2+"
PlayerInTurn"afk"

This is an example of a GameStateVector. Description is an human readable string - use it to promt the user. Round is the total numbers of rounds played. This number is incremented each time the the cards are shuffled. Step is the step in this round. 2 means that the players takes turn. Turn tells which actions the player in turn can take. 1 mean "Take an Action" and the "+" indicates that the player in turn haven't used his character ability. PlayerInTurn tells who is in turn - if it is you, you should properly not let the other players wait to long :-)

GameData

GameData
Name"Ultimate HotCit"
MinPlayers4
StarDistricts [
"BellTower": "Keep",
"Library": "Castle"
]
StarCharacters"Witch", "Abbot"
Passworduhc

This is an example of a GameData, which will be used to create a game. Name is a unique name for the game, that others players will use to join the game. MinPlayers is the minimum number of players that is needed before the game can start. MaxPlayers is omitted, which means that there is only the limit of number of characters (8 in this case). StarDistricts tells that one Keep District should be replaced by a Bell Tower, and a Castle should be replaced by a Library. StarCharacters states that the Witch and Abbot should be included in the game. This means that the Assassin will be replaced by the Witch and the Bishop by the Abbot. Password defines the password for the game. Joining players have to know this password and send it as described in "Join game".
